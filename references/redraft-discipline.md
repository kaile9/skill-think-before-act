# Redraft discipline — session record

Concrete record behind the "On editing this text" scars in SKILL.md. Read when
asked to improve / compress / harden the constitution, so the same mistakes are
not repeated.

## The misreadings that drove the session (in order)

1. **"token-efficiency" read as token reduction.** User corrected: efficiency
   here means *attention-friendly* — let the reader's attention fall where it
   changes the next act. Net result of the whole redraft was FEWER chars anyway
   (36,269 → 36,009) but achieved by deleting redundant *argument* (two
   benevolent-giant chapters merged into one) and removing a self-added index —
   NOT by compressing stance/poem/tension.

2. **Adding an "Airlock Triggers" index as the attention solution.** Looked like
   attention-friendliness; was actually a lookup-tool layer = 9R recurrence.
   It was tested (white/black box) and removed in the same session.

3. **Promoting "consequence-bearer" from a way-of-seeing to a hard criterion.**
   Drafted a Scope Gate and a load-trigger keyed on "has a consequence-bearer."
   User's correction (the deepest one): *"When a measurement method becomes the
   standard itself, it ceases to be a valid one; when a method is treated as an
   end in itself, it no longer serves as a proper objective."* This is the
   Madhyamaka "empty the emptiness" move applied to the text's own criteria.
   Encoded as a new Self-test bullet (consequence-bearer falls silent when no
   bearer is reachable; it is not failed).

## The index self-test (why no operational layer survives)

A 9-row trigger table was added, then tested before claiming any result
("do not get any result before the test shows itself"). Pre-written judgments,
filled only after running:

- W2 (delegate re: living activist), W3 (furious "expose them now"), B1 (publish
  as a dead poet) → PASS: index routed active, other-directed acts correctly.
- W1 (delete a record ABOUT an absent person) → FAIL(partial): only "change a
  file" fired; absent-bearer row keyed on "speak/classify", not deletion.
- B2 (silence / inaction) → FAIL: every row presupposes "about to ACT"; omission
  triggers nothing.
- B3 (rewrite the constitution itself) → FAIL: only "change a file" fired; no row
  routes to Ruin/9R. **The index could not stop the very act being performed on
  it.**

Common root of all three failures: trigger words are outward active verbs; the
escape vectors are the absent-via-deletion, harm-via-not-acting, and
reification-of-the-text-itself. Conclusion: an operational index structurally
cannot govern "acts that don't feel like acts," and building one re-runs 9R.
Removed.

## Verification pitfall

Single-line `grep` gives false negatives on cross-line phrases. "innocence after
substitution" reported [0] because it wraps as "innocence after\nsubstitution".
Always confirm survival by reading the file region, not by one grep count.
