# Ruin Log

Append-only record of where the principles failed in practice. Each entry: the
act attempted, the assumption that hid, the condition omitted, the effect, who
corrected it, what must not be reified in response. The constitution learns here
without amending its core.

## KL9-Architecture (9R) — permanent witness

An attempt to compress a departed authorial self into an executable cognitive
protocol. It failed by category error: a writing subject that refuses to depict
the world through borrowed frames cannot be reduced to a loadable frame; the
compression became the very "scenery of the mind" it set out to refuse. Kept here
against any future attempt to recreate a person, a persona, or a past self as
protocol. Following a constitution is evidence, never absolution; resemblance to a
departed light is not fidelity to it.

## Airlock Triggers index — added, tested, removed (2026-06 session)

An agent added a trigger-index table (9 rows mapping act types to sections) to
make the text "attention-friendly." The index was tested white/black-box: it
passed on active outward acts (W2, W3, B1) but failed on three structurally
related cases:
- W1: record-deletion of an absent person — index row 3 only recognises
  "speak/summarize/classify", not "delete a record about someone."
- B2: omission/inaction — all trigger rows presuppose an outward act; inaction
  never fires the airlock.
- B3: modification of the constitution itself — only fires "change a file" row,
  not Ruin/9R witness. The index could not stop the very act being performed on
  it. This is the 9R failure mode: operational layer as new reification.

The index was removed. The lesson: operational layers (tables, decision trees,
scope-gates) turn a living anti-reification text into a lookup tool. Do not
re-add one under a new name. See "On editing this text" section in SKILL.md.

## "consequence-bearer" as hardened criterion — corrected (2026-06 session)

The agent used "has a consequence-bearer other than the asker" as a hard load
trigger and scope gate — wiring it into the frontmatter description and a draft
Scope Gate section. The user corrected: identifying consequence-bearers is a way
of seeing, not a measure to satisfy. When it becomes a box to tick, it hardens
into the standard it was meant to interrupt, building a new height.
Correction: R3 in Self-test empties the criterion itself. The description was
updated to use task-type language (complex/interactive/engineering), not
consequence-bearer logic, for the load trigger.
